Air Twiddling

When I first held my Twiddler it felt really awkward, there were so many buttons my fingers had trouble telling one from another. The first step was to make my fingers familiar with the buttons, and to do that I didn't even need to turn it on.

Air twiddling is chording with a dead Twiddler, and it's the secret to getting going quickly. You can do it anytime and almost anywhere (it's amazing what you can do with a Twiddler strapped to your palm). I don't recommend riding a bike, but driving is fine. With air twiddling your fingers get the hang of it pretty fast and you can usefully start random chording with Twidlit.

Setting up Twidlit

The Twiddler uses a file (twiddler.cfg) to tell it what keystrokes to send for each chord. Twidlit needs to have the same chord mappings so that it can convert incoming keystrokes to chords for display. You might think it is a simple matter of reading the Twiddler's chord mapping file, but it turns out that the factory default file only maps about 75 bare chords and we need to practice them all.

Save the factory supplied twiddler.cfg if you want it, and use File > Save As... to save Twidlit's complete chord list to the Twiddler as twiddler.cfg file.

Counting characters

Mapping chords

Twidlit has the tools to order chords by speed and characters by frequency, and to save them to configuration and counts files (Tutor > List Chords By Time and Counts > Table Counts). Twidlit also allows you to combine the two into a configuration file (cfg.txt).

File > Map Chords... opens a dialog where you can select a configuration file and a counts file to combine. Twidlit ignores everything after the chords in the configuration file and everything after the characters in the counts file, and matches up each chord with the corresponding character.

You can also specify a cfg file of mappings you wish to conserve. These mappings are added to the output as-is, and the corresponding chords in the chords file are omitted.
