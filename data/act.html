Activities

Use Twidlit in any way that makes sense to you, but it was developed to support the following sequence of activities towards fast Twiddler fluency.
Getting familiar
Random chording
Mapping chords
Learning the mapping

Getting familiar

Air twiddling

When I first held my Twiddler it felt really awkward, there were so many buttons my fingers had trouble telling one from another. The first step was to get my fingers familiar with the buttons, and to do that I didn't even need to turn it on.

Air twiddling is chording with a dead Twiddler, and it's the secret to getting going quickly. You can do it any time and almost anywhere. It's amazing what you can do with a Twiddler strapped to your palm. Riding a bike is awkward, but driving is fine.

With air twiddling, your fingers get the hang of it pretty fast, and you can start really pushing your chording fluency with Twidlit.

Playing scales

If your Twiddler were a musical instrument you would expect to start out spending a lot of time playing scales. Random chording is Twiddler scales. You are not making music yet, but you are getting a feel for the chords: what's easy and what's hard, which side is which, and just how hard is it to coordinate three or four fingers?

Meanwhile, Twidlit keeps track of how long each chord takes. Don't worry if you are very slow or erratic to start with, it won't affect your times long-term.

Random chording

Timing all chords

There are 255 thumb-less chords. That's more than a whole keyboard's worth, with and without the Shift key pressed. A few are too awkward for frequent use, but there are enough left that once you are practiced with all of them you may choose to ignore some of the thumb-keys.

You need to find out for yourself which ones you are quick with and which are unmanageable, so random chording tests them all.

Initial setup

The Twiddler uses a file of chord mappings (twiddler.cfg) to tell it what keystrokes to send for each chord. Twidlit needs to have the same mappings so it can convert incoming keystrokes back to chords for the display.

Unfortunately, the Twiddler's factory-default mapping only includes about 75 thumb-less chords. To practice all of them you're going to need a different map file.

Twidlit comes set up with a complete mapping by default. If it has been changed you may not have all chords covered. In that case, File > Map All Chords... will be enabled. Use it to reinstall the complete mapping before saving to the Twiddler.

Plug in your Twiddler and use File > Save As CFG... to save Twidlit's current mapping to your Twiddler as twiddler.cfg. If you want to keep the factory-supplied twiddler.cfg you can rename it in the save dialog.

Which hand?

It is good to leave your dominant hand free for other things, but it's up to you. Maybe you want to be ambidextrous. Twidlit allows you to practice both hands and postpone the decision till you are ready. Separate times are kept for each hand.

Use Tutor > Left/Right Hand to specify the hand you are using. The progress bars on the Twiddler window flip to the side away from your palm.

Practice

Get as much practice as you can. Remember you don't need your brain for this. As with air twiddling, keep trying to find activities you can do at the same time (such as watching movies).

Twidlit keeps track of the last 16 presses of every chord and you can view them, or save them to a timed.chords file, using Tutor > List Chords by Time. You can use the file to generate a mapping.

Mapping chords

Unlike a conventional keyboard, the Twiddler is personal. There is no benefit to using the same mapping of chords to keystrokes as anyone else. Twidlit has the tools for you to develop the mapping that best suits you.

There are two conflicting imperatives in the design of a chord mapping. First and foremost is the simple rule: the more frequently a keystroke is used, the faster its chord should be, and both keystroke frequency and chord speed can be measured.

Fast and frequent

http://norvig.com/mayzner.html is the best resource I've seen for generic alphabetic character frequencies, but if you write code or just want to get a handle on punctuation frequencies then you need Twidlit's Counts facility to count your characters.

Emails, source code, even formatted documents saved as text, all contain a record of your (printable) keystrokes, and can be analyzed. Twidlit can save a count.keys file of keystrokes sorted by frequency that you can use to generate a mapping.

This is not a complete solution, there are plenty of keystrokes (such as the cursor keys) that don't leave a record. It can get things into proportion though, and I have found it invaluable for analyzing source code.

Mnemonic structure

Long-term, the less common keystrokes don't give your fingers enough practice to remember them without help. The second imperative is to use mnemonic structure to organize the chords memorably.

It makes little sense to copy someone else's mappings verbatim, but do copy their ideas, there are some good ones out there. Leave the fastest chords and most frequent keystrokes free though, and then use Twidlit to arrange them.

Putting it together

Using File > Create Map File you can select a hand-coded mnemonic cfg.chords file of less frequent chords, a timed.chords file of chords sorted by speed, and a count.keys file of keystrokes sorted by frequency, and combine them into a new cfg.chords file.

File > Assess Map File allows you to easily find the mappings in a cfg file that are far enough from the ideal to be worth reconsidering.

Chord Space Mining

There are 255 chords, but what patterns connect them? Chord patterns are a primary resource for devising mnemonics, but they are strongly interconnected. As you develop a mapping and chords get used up, only some of the chords for a given pattern will still be available.

How useful it would be to have a tool that lists the remaining available chords by pattern. It would then be much easier to find the pattern that best matches a group of keystrokes needing allocation. File > Group Chords is such a tool.

Learning the mapping

Put a few (maybe three) of the most frequent keystrokes in a practice keys file, and select it with Tutor > Keystrokes. The Text pane will prompt you with the keystroke(s), and if you can't remember the chord the Twiddler window will show it to you after a delay.

This is learning, not practice for speed. Start small and don't rush. Your fingers will learn whatever mapping you press, even if it is wrong, so be sure to keep getting it right every time.

Wait till you are confident before adding a couple more keystrokes, and leave the timer off till you are fully fluent. If you feel the need for speed then go back to polishing your random chording.
