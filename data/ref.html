Menus

File

Open...
Discards the current Twiddler configuration (if any) and uses the selected cfg.txt or cfg file.
Save As...
Opens a Save Text window showing the current chord mapping.
Reverse
Flips the buttons in each row
List Chords
Opens a Save Text window showing a list of every unmodified chord with a one keystroke mapping. This is the default mapping, and is ideal for chording practice and timing.
Map Chords...
Opens a dialog that allows you to create a new cfg.txt file that maps chords to characters.
Twiddler Settings...
Opens a dialog that allows you to modify the current configuration's Twiddler settings. These were used by Twiddler 2. I am not sure which ones affect Twiddler 3.
Preferences...
Allows you to select a location for the preference text files.

Counts

Include Bigrams
Whether to count bigrams (pairs of characters).
Include Ngrams
Whether to count the ngrams (sequences of characters) listed in TwidlitNGrams.txt.
Count File...
Allows you to select a file and add its character counts to the stored totals.
Count Files...
Allows you to select a folder and add the character counts of all contained files to the stored counts.
Set Range Displayed...
Opens a dialog where you can set the range of character frequencies displayed as table or graph.
Table Counts
Opens a Save Text window displaying the character frequencies as a table. The columns are ASCII, value, count and percent.
Graph Counts
Opens a Save Text window displaying the character frequencies as a graph. This is a simple text graph just to give a better indication of the relative magnitudes.
Clear Counts
Clears the counts to zero.

Tutor

Left/Right Hand
Sets the hand being used to twiddle. Hands are timed separately. (The progress bar will appear on the opposite side.)
Show Twiddler
Whether the Twiddler window is visible.
Highlight Expected Chord
Whether the Twiddler window highlights the expected chord.
Mark Chord Pressed
Whether the Twiddler window marks the chord that has just been pressed.
Speed...
Opens a dialog that allows you to adjust the time the Twiddler window waits for a chord to be pressed.
List Chords By Time
Opens a Save Text window showing a list of every unmodified chord with (and sorted by) the median time taken to press it. Initially these times are all zero.
Timed
Whether your chording (for the current hand) is timed.
Clear Times
Clears the chording times for the current hand.

Windows

Twiddler

The Twiddler window indicates the chord to press, while the progress bar to one side shows the elapsed time (up to six seconds). The chord is highlighted for two seconds or until pressed. If it isn't pressed it is highlighted again every six seconds.

The six second cycle time can be adjusted via the menu, and the component times can be changed individually in the preferences.

The window also marks any chord that is pressed, in yellow if it matches and black otherwise. Sometimes nothing appears when you thought you pressed a chord. This is the result of a miss-key. The Twiddler generates nothing if two or more buttons are pressed in the same row.

Ambiguity

If you like, you can map more than one chord to a keystroke. You might choose to map a Shift-less finger cord to A, for example. A can then be generated by this chord or by the chord for a plus the Shift thumb button.

This is a problem for Twidlit because it does not know which chord was pressed when an A arrives. The Twiddler window always displays the chord with the least number of thumb buttons. If there is still ambiguity it displays the chord with the least number of finger buttons.

Help

The help text contains a few hypertext links. Navigation buttons (back and forwards) appear at the top of the Help window if you follow an internal link.

Save Text

This window displays text that can be selected and copied, or saved. Editing is not supported.

You can use the Save As... button to save the contents. If it contains chord mappings the dialog allows you to save it as cfg.txt or cfg.

Chord files

Chord mapping files come in two flavours, cfg (binary) and cfg.txt (text). The Twiddler can only read cfgs, and you and I can only read cfg.txts. Happily Twidlit can read and write both.

I get confused trying to manage binary files that have to be given the same name (twiddler.cfg) when in use, so I keep my config files as text and only save them as binary when loading the Twiddler.

The Files of type selector below the file name box on the Open and Save As dialogs allows you to choose the type of file you want. No other extension can be used so you do not need to type the extension on the file name, it will be added if not present.

4finger syntax

Four-character tags represent chords, starting with the index finger. For example |-,'

| (pipe) is an inactive finger,
- (hyphen) is the middle button,
, (comma) is the button below and
' (apostrophe) is the button above.

Preferences

Much of the way Twidlit looks and works can be tuned by editing the preference text files. They are stored in Twidlit.jar, but you can access them by using File > Preferences... to choose a location to store them. Twidlit will copy any of them not already there to that location, and will read them from there next time it starts. Just delete any you don't edit.

TwidlitPreferences.txt

background.color 0xffffd9
The color of the background of most windows.
border.color 0xc0c0a0
The color of the border on the mapping dialog.
chord.pool.fraction 16
The fraction (1/x) of available chords in the sliding window from which the next chord is randomly chosen. A small value (eg 1) is undesirable as some chords will turn up more frequently than others. A large value (eg 255) makes chords appear in a fixed order.
control.separation.size 10
The size of the separation between controls on the Twiddler Settings windows.
count.repeats.max 2
When character frequency counting, any more are ignored after this many of the same one in a row.
count.clear.confirm true
Whether to open a dialog to confirm you want to clear the character frequency counts.
key.wait.msec 80
The milliseconds Twidlit waits for a further keystroke to arrive from the Twiddler before converting the accumulated keys into a chord.
new.line windows
The new line style used in text files.
twiddler.background.color black
The background color of the Twiddler window.
twiddler.button.color lightGray
The color of the buttons and progress bar in the Twiddler window.
twiddler.highlight.color red
The color used to highlight the buttons of a chord in the Twiddler window.
twiddler.label.color white
The color of the thumb button labels in the Twiddler window.
twiddler.mark.match.color yellow
The color used to mark a chord that matches the one highlighted.
twiddler.mark.mismatch.color black
The color used to mark a chord that does not match the one highlighted.
twiddler.progress.step.msec 150
The time in milliseconds between updates of the Twiddler window progress bar.
twiddler.highlight.msec 2000
The milliseconds for which a chord is highlighted in the Twiddler window.
twiddler.rehighlight.msec 4000
The milliseconds delay before a chord is highlighted again in the Twiddler window. The progress bar duration is the sum of this and the highlight time.
twiddler.mark.msec 2000
The milliseconds for which a chord is marked in the Twiddler window.
window.border.size 8
The size of the border on some windows.
write.syntax 4finger
Whether to use 4finger or 0MRL when writing cfg.txt files.

TwidlitKeyValues.txt, TwidlitKeyNames.txt and TwidlitKeyEvents.txt

Twidlit is set up to use the US keyboard mapping. If yours is different you can probably find it in www.w3.org/TR/DOM-Level-3-Events-code and change these files.

TwidlitUnprintables.txt

Some ASCII values are unprintable (backspace, for example). When every keystroke needs to be visible the values in this file are treated specially.

TwidlitDuplicates.txt

Some keys are duplicates (Numpad4, for example). You can configure your twiddler to generate them but they never appear in text.

TwidlitLost.txt

Some keystrokes never reach an application (<Alt><Tab></Alt>, for example). They vary with OS.
